### 操作一：内容分类与文件迁移

**判定标准：** 判断一个文件应归类为"读书笔记"还是"结构化框架工具"？

| 标志 | → 读书笔记（06-reading-notes/） | → 工具框架（06-工具与框架/） |
|------|-------------------------------|--------------------------|
| 内容类型 | 原文摘录+个人批注 | 方法论/对比表/操作流程 |
| 组织形式 | 按原书章节组织 | 按主题/维度重新组织 |
| 写作风格 | 记录向（叙述+引述） | 工具向（步骤+表格+清单） |
| 是否可独立做判断 | 否（依赖原文上下文） | 是（自包含的决策框架） |
| 案例 | 《量价分析》读书笔记 | 看盘技术与交易策略综合 |

**移动执行步骤：**

```python
# 1. 复制文件到目标目录
import shutil
shutil.copy2(src_path, dst_path)

# 2. 更新被移动文件内部的 wikilink
#    （原路径如 [[../06-随堂笔记/看盘记录汇总]] → [[看盘记录汇总]]）
new_content = content.replace(
    "[[../06-随堂笔记/看盘记录汇总|看盘实践]]",
    "[[看盘记录汇总|看盘实践]]"
)

# 3. 更新引用该文件的其他文件中的链接
#    （原路径如 [[../06-工具与框架/看盘_《量价分析》]] → [[../../06-reading-notes/看盘_《量价分析》]]）

# 4. 删除源文件
os.remove(src_path)

# 5. 更新源目录的 README（移除已迁移条目）
# 6. 更新目标目录的 README（如需）
```

### 操作三：重要内容上网比对更新

**适用场景：** wiki 中的规则性/时效性较强的内容（上市规则、行业数据、政策要求）

**执行步骤：**

```
Step 1: 识别需核查的内容要素
  从当前 wiki 文件中提取可验证的具体数据点：
  - 门槛/标准/数字（如"净利≥1500万"）
  - 时效表述（如"近2年"、"最新"）
  - 制度/政策名称（如"四套标准"）

Step 2: 搜索验证
  用 DuckDuckGo (ddgs) 搜索，region='cn-zh' 获取中文结果
  每个关键数据点至少比对 2-3 个独立搜索结果

Step 3: 差异分析与更新
  搜索结果与 wiki 不一致时：
  - 搜索结果的可靠性 > 现有wiki（因为wiki来自个人收藏，可能已过时）
  - 标注更新时间戳（如 "updated: 2026-05-24"）
  - 添加数据来源标注（如 "数据来源：人民网2026年3月"）

Step 4: 补充新发现
  搜索过程中发现 wiki 中没有的重要新信息 → 一并补充
  如：创业板正式启用第三套标准（允许亏损企业）、大基金三期动态等
```

## 技能文件写入注意事项

写入 `~/.hermes/profiles/ai-investor/skills/` 下的 SKILL.md 时，`patch` 工具同样可能因路径解析失败。替代方案：

```python
import os
home = os.path.expanduser("~")
path = os.path.join(home, ".hermes", "profiles", "ai-investor", "skills", "<skill-name>", "SKILL.md")
with open(path, "r", encoding="utf-8") as f:
    content = f.read()
# ... 做 content.replace() 或 re.sub() ...
with open(path, "w", encoding="utf-8") as f:
    f.write(content)
```

`skill_manage(action='patch')` 是另一个可用的替代方案（本工具链白名单内可用）。
