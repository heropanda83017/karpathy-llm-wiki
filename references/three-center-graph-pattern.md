# 三中心图谱模式 — Obsidian 图谱优化方案

> 用于多方向知识库的图谱可视化，避免 index.md 成为单一中心导致杂乱的星爆图。

## 问题

Obsidian 图谱以 `[[wikilink]]` 为边。当 index.md 直接链接所有页面时，图谱呈现为：
```
index.md ──→ 所有页面（单个中心，30+ 辐射线，杂乱）
```

## 解决方案：三层中心结构

```
index.md ──→ 三个中心 ──→ 各自页面集群
              中心 ←──→ 页面回链
              中心 ←──→ 中心互链
```

### 三层角色

| 层级 | 文件 | 出链目标 | 入链来源 |
|------|------|---------|---------|
| L1 入口 | `index.md` | 仅链接三个中心 + 共用工具页 | 无 |
| L2 中心 | `🏗️ 理想之地实战中心.md` | 本方向所有内容页 + 其他中心 | 内容页回链 |
| L3 内容 | `05-applications/xxx.md` | 回链到所属中心 | 中心页的出链 |

### 创建步骤

```
Step 1: 确定中心划分
  按知识库的实际使用方向划分。示例：
  - 🏗️ 理想之地实战中心 → 05-applications, 04-tools(Prompt), 07-practices(技能卡)
  - 🤖 AI基础学习中心   → 01-theory, 02-fundamentals, 03-core-ai, 06-reading-notes
  - 📈 金融投资研究中心  → 08-investment(6个子目录)

Step 2: 创建中心页面
  每个中心页包含：
  - 本方向的简短定位说明
  - 所有内容页的 [[wikilink]] 列表（按子目录分组）
  - 跨中心链接：[[other_center]]（注意：不链回 index，避免图谱星爆）

Step 3: 为内容页添加回链
  遍历所有内容页，在文件末尾追加：
  ## 关联
  - [[所属中心]]
  注意：04-tools 和 07-practices 等共用工具不加中心链接，
  它们在图谱中作为独立节点自然连接三个集群。

Step 4: 简化 index.md
  index.md 只保留：
  - 三个中心的 [[wikilink]]
  - 共用工具/方法的 [[wikilink]]（04-tools/07-practices/总纲）
  - 删除对单个内容页的直链
```

### 图谱效果

```
index.md ─── 🏗️ 实战中心 ─── 05-applications (2个文件)
              🤖 AI学习中心  ─── 01-theory (2) + 02-fundamentals (3)
                                + 03-core-ai (4) + 06-reading-notes (5)
              📈 投资中心    ─── 08-investment (6个文件)
              
              04-tools/  ─── (3个文件，共享工具)
              07-practices/ ── (5个文件，共享方法)
```

### 验证清单

```
☐ 全库无 [[index]] 回链（re.findall 确认，index 保持只出不进的纯入口）
☐ index.md 只链接三个中心 + 共用工具，不直链单个内容页
☐ 每个中心页链接了本方向所有内容页
☐ 每个内容页回链到所属中心（re.findall 确认）
☐ 三个中心互相链接（跨中心导航）
☐ 共用工具页没有中心回链（保持中立节点）
☐ SCHEMA.md 更新了目录结构描述
☐ log.md 追加了本次变更记录
```

### 适用范围

| 适用 | 不适用 |
|------|--------|
| 多方向知识库（3-5 个学习/研究方向） | 单一主题 wiki（1 个方向） |
| 页面数量 20+，需要分组导航 | 小型 wiki（<10 页） |
| 用户使用 Obsidian 图谱功能 | 用户只用文件列表浏览 |
