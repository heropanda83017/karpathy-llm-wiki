# AIGC-KB Vault 审计实战记录

> 针对学习笔记型 vault（以文件夹组织为主，wikilink稀疏）的优化方法论。

## 审计方法论

### 1. 目录结构审计

```
关键检查项：
- 编号是否连续？有无重复编号？（如两个01目录）
- 空目录是否需要删除？
- 目录名是否准确反映内容？
```

### 2. 内容重叠检测

```python
# 提取两页正文（去掉frontmatter）
# 提取所有2字以上中文词
words = re.findall(r'[\u4e00-\u9fff]{2,}', content)
# 计算重叠率 = len(词A ∩ 词B) / len(词A)
```

**阈值参考：**
- <10% → 差异大，不合并
- 10-30% → 上下游关系，加强双向链接
- >30% → 考虑合并

### 3. 合并策略

| 条件 | 处理方式 |
|------|---------|
| <500字 + 内容不完整 | 直接删除 |
| 内容薄但有价值 | 合并到关联页面作为附录 |
| 主题相近 | 合并为系列，分章节 |
| 标题与内容不符 | 修正标题或删除 |

### 4. 编号修复

遇到编号冲突（如两个01目录）或断号时：
```
01-theory/     → 保留
01-fundamentals/ → 02-fundamentals (解决冲突)
02-core-ai/   → 03-core-ai (顺延)
03-tools/     → 04-tools (顺延)
```

### 关键经验

- 学习笔记型 vault 的孤立页面是正常现象，不应视为问题
- 合并的收益：10篇→3篇，找东西快3倍，内容零损失
- 合并前先读内容判断主题相关度，不强行合并无关页面
