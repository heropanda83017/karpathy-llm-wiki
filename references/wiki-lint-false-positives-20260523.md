# wiki_lint 假阳性与死引用修复实录（2026-05-23）

## 背景

对理想之地 vault（46页，E:/Landofdream/wiki-land-of-dream-planning/）做 wiki_lint 时发现的假阳性模式和修复经验。

## 假阳性类型

### 类型1：模板文本 `[[wikilinks]]`

**位置**：`log.md`、`SCHEMA.md`
**原因**：这两个文件包含 Markdown 格式教程文本，其中的 `[[wikilinks]]` 是语法示例，不是真实链接。
**处理**：在 lint 的链接完整性检查中跳过这两个文件，或者过滤掉链接目标为 `wikilinks` 的条目。

### 类型2：锚点链接 `[[#section|display]]`

**位置**：`concepts/8-service-blocks.md` L92
**原因**：同一页面内的锚点跳转 `[[#01 商业运营 + 人才公寓|见商业运营最新进展]]`，`split('#')[0]` 后得到空字符串，被误判为 `[[]]`。
**处理**：链接检查中，若 `link.split('|')[0].split('#')[0].strip()` 为空 → 跳过（是锚点链接）。

### 类型3：入链/出链计数中的不可见链接

**位置**：全 vault 通用
**原因**：某些页面引用了不存在的页面，但 lint 的孤立页面判定 `(0出链, 0入链)` 漏掉了只有死出链的页面。`format-errors.md`、`weekly-report-*.md` 等被删除页面会产生死出链，但出链计数时若过滤了死链接，这些页面反而显示为 `0出链`。
**处理**：出链计数应包含所有 `[[link]]`（包括指向不存在的页面），因为这些链接本身也是内容的一部分。孤立页面的判定标准改为：
```
孤立 = 无出链(含死链) AND 无入链
单向 = 有出链(含死链) BUT 无入链
```

## 死引用修复经验

### 修复案例：index.md L73

**旧**：`[[weekly-report-2026-W19]] · [[competitor-landscape]]`
**问题**：`weekly-report-2026-W19.md` 页面已删除
**修复**：`[[competitor-landscape|竞品监测]]`
**原则**：不要删除整行。保留行上下文，只把死链接替换为最近的存活页面。

### 修复检查清单

1. 确认页面确实已删除（不是大小写/路径问题）
2. 找到替换页面——内容最接近的现存页面
3. 如无合适替换，只删除 `[[link]]` 部分，保留前后的文字
4. 修复后二次 lint 确认死引用归零
