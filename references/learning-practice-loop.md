# 学习实践闭环工作流

> 从理想之地项目实战中提炼的学习模式：学AI概念 → 在工作中应用 → 写成实战笔记固化。

## 核心理念

传统学习：看书→记笔记→（然后就没有然后了）
本模式：学习→在实际项目中用→把经验写成笔记→下次用的时候直接翻笔记

## 双循环架构

```
工作循环（每天）
  ① 源文件/ → 读取数据 + 新资料
  ② 生成成果到 输出/ + 更新 wiki
  ③ 稳定后归档到 源文件/ + 清理输出

学习循环（每完成一个技术性任务触发）
  ① 复盘：刚才用到了什么AI知识？
  ② 提炼：写成一篇实战笔记 → AIGC-KB/07-practices/
  ③ 归档：更新 index.md
```

两条循环的节奏不同：工作循环是每天跑，学习循环是"每完成一个有技术含量的任务就写一篇"。

## 笔记模板

每次写实战笔记时按以下结构：

```
# 标题（从具体任务提炼的抽象概念）
## 一、场景描述（什么任务、为什么需要这个知识）
## 二、关键概念（200字以内说清楚是什么）
## 三、实际案例（从真实工作流中截取，附代码/配置/截图）
## 四、经验总结（踩了什么坑、怎么解决的、下次怎么做）
## 五、关联阅读（AIGC-KB中的其他相关笔记）
```

## 适用条件

不是每个任务都值得写笔记。触发条件：

| 触发条件 | 例子 |
|---------|------|
| 用到了一个新的AI概念/工具 | 第一次用bb-browser采集小红书 |
| 踩了一个有价值的坑 | Edge CDP登录态管理 |
| 用户纠正了写作/分析方式 | 不要用-连字符、追加模式 |
| 完成了一个完整的工作闭环 | 三歩闭环从读取到归档 |
